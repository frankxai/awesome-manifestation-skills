# Evidence Log

Keep the receipts. The log is what turns "I think this is working" into something you can actually read. It also keeps the practice honest: by separating what you *did* from what *surfaced*, you can see real progress without crediting magic — and catch survivorship bias before it fools you.

## When to use

The user wants to track a manifestation or goal practice, journal daily progress, or review whether the practice is doing anything.

## Daily entry

Each evening, capture one line per relevant event:

```
[YYYY-MM-DD]
- [what moved toward the vision today]  ·  source: made | surfaced  ·  next: [optional]
```

- **made** = you caused it (you sent the email, did the rep, shipped the thing).
- **surfaced** = it appeared (a relevant opportunity, intro, or idea showed up). This is the attention-filter at work — note it, don't mysticize it.

## Steps

1. **Prompt for today.** Ask what moved toward the vision — even small. If nothing, log "no movement" honestly; blank days are data too.
2. **Tag the source.** For each item, mark made vs. surfaced. Don't let "surfaced" items get credited as proof of a cosmic mechanism — they're evidence the filter is working.
3. **Weekly review.** Once a week, read the last seven days. Surface: the trend (more movement or less?), how many actions the user took, and how many relevant things surfaced. Report direction, not noise.
4. **Adjust one thing.** Based on the review, suggest one change to the practice — not five.

## Output

The appended daily entry, and on review days a short summary: trend · count of made vs. surfaced · one suggested adjustment.

## Boundaries

- Stay honest. Don't inflate coincidences into proof. The value is an accurate picture over time.
- Don't diagnose or interpret distress. This is a progress log, not a clinical tool.
