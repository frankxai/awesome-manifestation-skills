# Contributing to Awesome Manifestation Skills

Thanks for helping build the best grounded resource for manifestation practice and the systems around it. This list has a stance: **keep the mechanisms, drop the cosmology.** Contributions are held to that bar.

## What belongs here

- **Tools & apps** that help with state-setting (music, soundscapes), imagination/mental rehearsal, attention, or goal-setting — ideally with a developer API or agent use case.
- **Agent skills** (`SKILL.md`) that run a grounded practice.
- **Frameworks** for making goals specific and checkable (WOOP, implementation intentions, OKRs).
- **AI creation tools** for externalising a vision (image, music, page, plan).
- **Research** — peer-reviewed where possible — on mental rehearsal, attention, affect, music psychology, or goal-setting.

## What doesn't belong here

- Anything that presents metaphysical claims as science ("quantum manifestation", "frequency of the universe" as physics).
- Outcome guarantees, income promises, or "the universe will provide" framing.
- Anything implying people attract their own misfortune.
- Dead links, unmaintained projects, or pure affiliate spam.

## Format for list entries

```markdown
- **[Name](https://url)** — Short description of what it does and why it's useful, grounded and specific. `Language` (if a library)
```

## Adding an agent skill

1. Create `skills/<skill-name>/SKILL.md`.
2. Frontmatter must include `name` and a `description` that states **when** to use the skill (trigger conditions matter — that's how an agent routes to it).
3. Keep the practice grounded: name the mechanism, include honest boundaries, and route distress/clinical content to real support.
4. Add a one-line entry under [Agent Skills in This Repo](README.md#agent-skills-in-this-repo).

## Submitting

Open a pull request with a clear title and a one-line rationale. Keep sections alphabetical-ish within reason, and put the most load-bearing entries first. One PR per logical change.
